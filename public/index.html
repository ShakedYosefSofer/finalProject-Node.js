Api toys docs from public

# HTTP Method Route Description Parameters / Request Body
GET / display this page home page
http://localhost:3001/
1 GET domain/toys all list toys skip (optional): page number to return (default: 1).
http://localhost:3001/toys/?skip=1
skip (optional): number of toys per page (default: 1, maximum: 10)
http://localhost:3001/toys/?skip=6
sort (optional): field to sort by (default: price).
http://localhost:3001/toys/?skip=1&sort=price
reverse (optional): if set to yes, sorts in descending order.
http://localhost:3001/toys/?skip=1&sort=price&reverse=yes
2 GET domain/toys/price Returns a list of toys filtered by price range. min (optional): minimum price to filter by (default: 0). http://localhost:3001/toys/price?min=100
max (optional): maximum price to filter by (default: maximum 1000). http://localhost:3001/toys/price?min=10&max=20
3 GET toys/search?s=search name or category Returns a list of toys matching a search query. search?s="": the search query to match against toy names or descriptions. http://localhost:3001/toys/search?s=doll
4 GET /toys/category/:catName Returns a list of toys by category catName: the name of the category to retrieve toys for. http://localhost:3001/toys/category/LEGO
5 GET /toys/single/:id Returns a single toy by its ID id: the ID of the toy to retrieve. http://localhost:3001/toys/single/65fb544133c337293963db1d
6 POST /toys Adds a new toy.
Token is required!		 Request Body: Toy data to be added in the following format.
price: 1-1000
img_url: is allowed and not required 
{
name: String,
info: String,
category: String,
img_url: String,
price: Number
 }
7 PUT /toys/:idEdit Updates an existing toy
Token is required!		 idEdit: the ID of the toy to update.
Request Body: Toy data to be updated in the following format.
price: 1-1000
img_url: is allowed and not required 
{
name: String,
info: String,
category: String,
img_url: String,
price: Number
 }
8 DELETE /toys/:idDel Deletes a toy by its ID
Token is required!		 idDel: the ID of the toy to delete.






Users API

# HTTP Method Route Description Request Body
1 GET /users Test endpoint to check if the server is running N/A
2 GET /users/userInfo Get the information for the authenticated user (requires authentication) N/A
3 POST /users Create a new user { name: String, email: String, password: String }
4 POST /users/login Authenticate a user and receive a JWT token in response { email: String, password: String }






Users/Admin Token Abillities


*** When signing up, your default status will be "user" ***



# Admin Default User
1 Has the ability to add new toys to the system Has the ability to add new toys to the system
2 Can edit all information related to any toy in the system Can only edit information related to toys they have created or are associated with their account
3 Has the ability to delete all toys from the system Can only delete toys that they have created or are associated with their account
